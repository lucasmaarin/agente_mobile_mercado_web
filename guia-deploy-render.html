Agente Mobile Mercado no Render

Desenvolvido para configurar o projeto a partir do repositório GitHub da Mobile Mercado

📋
O que você vai fazer

Neste guia você vai colocar o Agente Mobile Mercado no ar usando o Render — uma plataforma que hospeda o sistema na internet. Não precisa saber programar. Siga cada passo na ordem.

⚠️ Antes de começar
Você vai precisar das credenciais do projeto (chaves de acesso). Peça ao Lucas antes de iniciar — ele vai te enviar uma lista de valores para copiar e colar. Sem essas credenciais não é possível concluir o deploy.

1
Criar conta no Render
  • 1
    Abra o navegador e acesse: render.com
  • 2
    Clique no botão Get Started for Free (ou Sign Up).
  • 3
    Recomendado: clique em Continue with GitHub — isso facilita a conexão com o repositório nas próximas etapas. Se ainda não tiver conta no GitHub, crie em github.com primeiro.
  • 4
    Confirme seu e-mail se o Render pedir. Após isso, você estará no painel principal.
✅ Como saber que deu certo
Você vê a tela do painel do Render com a opção de criar um novo serviço.
2
Conectar o GitHub da Mobile Mercado

O código do sistema está em um repositório no GitHub da Mobile Mercado. Você precisa dar permissão para o Render acessar esse repositório.

  • 1
    No painel do Render, clique em + New (canto superior direito) e depois em Web Service.
  • 2
    Na tela que aparecer, clique em "Connect a repository" e depois em Connect GitHub.
  • 3
    Uma janela do GitHub vai abrir pedindo autorização. Clique em Authorize Render.
  • 4
    O GitHub vai perguntar em quais repositórios o Render pode ter acesso. Escolha "Only select repositories" e selecione o repositório agentemercado (ou o nome exato que o Lucas informar). Depois clique em Install & Authorize.
  • 5
    De volta ao Render, o repositório vai aparecer na lista. Clique em Connect ao lado dele.
3
Configurar o serviço web

Após conectar o repositório, o Render vai mostrar uma tela de configuração. Preencha os campos conforme abaixo:

Name
agentemercado
Region
South America (São Paulo) — se disponível, ou US East (Ohio)

main

Node

npm install && npm run build

⚠️ Atenção no plano
O plano Free do Render deixa o serviço "dormir" quando ninguém acessa, causando lentidão. Use o plano Starter para o sistema ficar sempre disponível.

Ainda não clique em "Create Web Service". Primeiro configure as variáveis de ambiente na seção abaixo.

4
Adicionar as variáveis de ambiente

As variáveis de ambiente são as "chaves secretas" do sistema — elas conectam o agente ao banco de dados, à inteligência artificial e às notificações. Sem elas, o sistema não funciona.

🔑 O que você precisa pedir ao Lucas
Peça ao Lucas que te envie os valores de cada variável listada abaixo. Ele tem acesso ao Firebase e à OpenAI. São textos longos — copie e cole exatamente como ele enviar, sem espaços extras no início ou fim.

Onde adicionar no Render

  • 1
    Na tela de criação do serviço, role a página para baixo até encontrar a seção "Environment Variables" (ou clique na aba Environment se o serviço já existir).
  • 2
    Para cada variável da tabela abaixo: clique em Add Environment Variable, cole o nome no campo Key e o valor no campo Value.
  • 3
    Repita para todas as variáveis obrigatórias.

Lista de variáveis (peça os valores ao Lucas)

💡 Dica ao copiar e colar
Copie cada valor sem clicar duas vezes nele — clique uma vez no começo do texto, segure Shift e clique no final, depois Ctrl+C. Evite copiar espaços invisíveis no início ou fim do valor.
5
Fazer o deploy e verificar
  • 1
    Depois de preencher todas as variáveis, role até o final da página e clique em Create Web Service.
  • 2
    O Render vai começar a instalar o projeto automaticamente. Você verá um painel com logs (textos rolando). Isso pode levar de 3 a 10 minutos.
  • 3
    Aguarde aparecer a mensagem "Your service is live 🎉" ou o status Live no topo da página.
  • 4
    O Render vai gerar uma URL para o sistema, parecida com:
    https://agentemercado.onrender.com
    Clique nessa URL e verifique se o sistema abre corretamente.
⚠️ Se aparecer erro nos logs
Se o deploy falhar (aparecer em vermelho), as causas mais comuns são:

1. Uma variável de ambiente foi digitada errada (nome ou valor com espaço).
2. O valor de uma variável obrigatória ficou em branco.

Nesse caso, clique na aba Environment, revise as variáveis e clique em Manual Deploy → Deploy latest commit para tentar novamente. Se persistir, avise o Lucas com uma captura de tela dos logs em vermelho.
✅ Deploy funcionando
O sistema estará no ar e acessível pela URL gerada pelo Render. Qualquer atualização feita pelo Lucas no GitHub será aplicada automaticamente.
6
Domínio personalizado (opcional)

Se a Mobile Mercado tiver um domínio próprio (ex: agentemercado.com.br), você pode configurá-lo para substituir a URL padrão do Render.

  • 1
    No painel do serviço no Render, clique na aba Settings.
  • 2
    Role até "Custom Domains" e clique em Add Custom Domain.
  • 3
    Digite o domínio (ex: www.agentemercado.com.br) e siga as instruções do Render para apontar o DNS. Essa parte pode precisar de ajuda do Lucas ou da empresa de registro do domínio.

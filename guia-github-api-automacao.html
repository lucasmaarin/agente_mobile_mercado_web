O que é a Automação de Implantação

A Automação de Implantação é um painel administrativo que permite adicionar, editar e remover estabelecimentos (supermercados) do sistema sem precisar mexer em código. Quando você adiciona um estabelecimento pelo painel, ele modifica automaticamente um arquivo no GitHub e o sistema se atualiza sozinho.

Para isso funcionar, o painel precisa de permissão para escrever nesse arquivo. Essa permissão é concedida por meio de uma chave de acesso da API do GitHub chamada Personal Access Token.

📌 Resumo do que você vai fazer
1. Gerar um Token de Acesso no GitHub (chave de permissão).
2. Anotar os dados do repositório (dono e nome do repositório).
3. Configurar tudo nas variáveis de ambiente do Render.

2
Gerar o Personal Access Token no GitHub

O Personal Access Token (PAT) é uma senha especial que permite ao painel administrativo modificar arquivos no GitHub automaticamente. Você vai gerar esse token na conta do GitHub da Mobile Mercado.

🔒 Segurança — leia antes de continuar
O token gerado aparece uma única vez. Depois de fechar a página do GitHub, não é possível ver o token novamente — apenas criar um novo. Copie e cole imediatamente após gerar.

Caminho para chegar até a geração do token:

Foto de perfil (canto direito) → Settings → Developer settings → Personal access tokens → Tokens (classic)
  • 1
    Acesse github.com e entre na conta da Mobile Mercado.
  • 2
    Clique na foto de perfil no canto superior direito → depois em Settings.
  • 3
    Role até o final do menu à esquerda e clique em Developer settings.
  • 4
    Clique em Personal access tokens → depois em Tokens (classic).
  • 5
    Clique em Generate new token → depois em Generate new token (classic).
  • 6
    Preencha os campos conforme abaixo:
Note (nome do token)
automacao-implantacao
Expiration (validade)
No expiration — sem expiração (ou 1 year)
Permissões a marcar
✓ repo — Full control of private repositories
(marcar o item "repo" pai seleciona automaticamente todos os sub-itens)
  • 7
    Clique em Generate token no final da página.
  • 8
    O GitHub vai mostrar o token gerado — uma sequência longa de letras e números começando com ghp_. Copie agora! Use o botão de cópia ao lado do token.
✅ Token gerado com sucesso
Guarde o token em local seguro (ex: bloco de notas fechado com senha). Ele vai para a variável GITHUB_TOKEN no Render.
3
Encontrar os dados do repositório

Além do token, o painel precisa saber em qual repositório do GitHub ele deve escrever. Essas informações estão na URL do repositório no GitHub.

  • 1
    Ainda logado no GitHub, clique em seu perfil → Your repositories.
  • 2
    Abra o repositório da Automação de Implantação (o Lucas vai te dizer o nome exato).
  • 3
    Olhe a URL do navegador. Ela vai ser parecida com:
    https://github.com/lucasmaarin/automacao_implantacao
    A parte em vermelho é o GITHUB_OWNER.
    A parte em azul é o GITHUB_REPO.
4
Configurar as variáveis no Render

Agora você vai entrar no painel do Render do serviço de Automação de Implantação e adicionar as variáveis de ambiente.

  • 1
    Acesse render.com e entre na sua conta.
  • 2
    Clique no serviço painel-admin-agente-mobile (ou o nome que aparecer).
  • 3
    Clique na aba Environment no menu do serviço.
  • 4
    Para cada variável da tabela abaixo, clique em Add Environment Variable e preencha o Key (nome) e o Value (valor).
  • 5
    Após adicionar todas, clique em Save Changes.

Variáveis obrigatórias

Nome (Key) Valor (Value) Obrigatório?
GITHUB_TOKEN O token gerado na Seção 2 (começa com ghp_) SIM
GITHUB_OWNER Nome de usuário do GitHub da Mobile Mercado (ex: lucasmaarin) SIM
GITHUB_REPO Nome do repositório do agente (ex: agente-mobile-mercado) SIM
GITHUB_BRANCH main padrão: main
GITHUB_FILE_PATH src/config/dominios.ts padrão já definido
ADMIN_PASSWORD Senha de segurança do painel — defina uma senha forte e anote SIM
RENDER_URL URL do agente no Render (ex: agente-mobile.onrender.com) SIM
NODE_VERSION 20 já definido no render.yaml
ℹ️ Sobre GITHUB_FILE_PATH e GITHUB_BRANCH
Essas variáveis já têm valores padrão definidos no código — você não precisa adicioná-las a menos que o Lucas indique um caminho ou branch diferente. Se não souber, deixe de fora.
5
Variáveis opcionais (WhatsApp e E-mail)

O painel pode enviar automaticamente instruções de configuração de DNS por WhatsApp e E-mail para os novos estabelecimentos. Essa funcionalidade é opcional e requer contas nas plataformas Twilio e Resend. Peça ao Lucas se devem ser configuradas.

WhatsApp via Twilio (opcional)

Nome (Key)O que é
TWILIO_ACCOUNT_SID ID da conta Twilio (começa com AC...)
TWILIO_AUTH_TOKEN Senha de autenticação da conta Twilio
TWILIO_WHATSAPP_NUMBER Número do WhatsApp Twilio (ex: whatsapp:+14155238886)
WHATSAPP_ENABLED true para ativar / false para desativar

E-mail via Resend (opcional)

Nome (Key)O que é
RESEND_API_KEY Chave de API do serviço Resend (para envio de e-mails)
EMAIL_FROM E-mail remetente (ex: contato@mobilemercado.com.br)
⚠️ Se não configurar agora
O painel funciona normalmente sem WhatsApp e E-mail. Essas funcionalidades podem ser adicionadas depois. Sem elas, os estabelecimentos são cadastrados normalmente, apenas sem o envio automático de instruções.
6
Verificar se funcionou
  • 1
    Após salvar as variáveis, vá na aba Logs do serviço no Render.
  • 2
    Clique em Manual Deploy → Deploy latest commit para reiniciar o serviço com as novas variáveis.
  • 3
    Aguarde o status ficar Live.
  • 4
    Acesse a URL do painel (gerada pelo Render para esse serviço) e tente listar os estabelecimentos. Se a lista carregar (mesmo que vazia), a conexão com o GitHub está funcionando.
❌ Se aparecer "Configurações do GitHub incompletas"
Isso significa que uma ou mais variáveis estão em branco ou com erro de digitação. Volte à aba Environment, verifique GITHUB_TOKEN, GITHUB_OWNER e GITHUB_REPO e refaça o deploy.
❌ Se aparecer "Arquivo não encontrado no repositório"
O GITHUB_REPO pode estar errado (nome incorreto do repositório) ou o arquivo src/config/dominios.ts não existe nesse repositório. Confirme o nome exato com o Lucas.
✅ Tudo certo quando...
A lista de estabelecimentos carrega e você consegue adicionar um estabelecimento de teste pelo painel. O GitHub vai mostrar um novo commit automático no repositório do agente — confirmação de que a integração está funcionando.
